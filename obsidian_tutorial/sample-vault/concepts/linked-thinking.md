# Linked Thinking

> A methodology for building knowledge through connections, not hierarchies.

## What Is Linked Thinking?

Linked thinking is the practice of connecting ideas through explicit links rather than filing them into rigid folder hierarchies. Instead of asking "where does this go?", you ask "what does this connect to?"

## Core Principles

1. **Links over folders** — One note can connect to many others; a file can only be in one folder
2. **Emergent structure** — Organization emerges from connections, not pre-planned categories
3. **Bidirectional linking** — Every link creates a two-way relationship (forward link + backlink)
4. **Atomic notes** — Each note covers one concept, making connections clearer

## Comparison with Folder-Based Organization

| Folder-Based | Link-Based |
|---|---|
| "Where does this go?" | "What does this connect to?" |
| One location per note | Unlimited connections |
| Structure decided upfront | Structure emerges over time |
| Rigid | Flexible |
| Good for: executable files, templates | Good for: ideas, research, projects |

## Practical Example

Instead of:
```
projects/active/project-alpha.md
```

Use links within the note:
```markdown
#tags: [project, active]
# Links: [[Customer Dashboard]], [[Q3 Planning]], [[Alice Chen]]
```

Now [[Project Alpha]] is discoverable from multiple angles — by project status (#active), by topic ([[Customer Dashboard]]), by planning context ([[Q3 Planning]]), and by team member ([[Alice Chen]]).
