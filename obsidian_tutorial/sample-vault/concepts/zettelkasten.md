# Zettelkasten

> "One note per idea, densely linked."

## Origin

Developed by German sociologist Niklas Luhmann (1927–1998). He used a physical slip-box (Zettelkasten = "note box") with over 90,000 index cards, all interconnected. It enabled his prolific output: 70 books and 400+ scholarly articles.

## Core Principles

### 1. Atomicity
Each note contains exactly ONE idea. Not one topic — one idea. If a note has two ideas, split it.

### 2. Connectivity
Every note links to at least one other note. No orphan notes. The value of a note is proportional to its connection count.

### 3. Your Own Words
Always write notes in your own words. Copy-pasting is anti-Zettelkasten. The translation process is where understanding happens.

### 4. Organic Growth
No predefined categories. Structure emerges from the bottom up as you add and connect notes.

## Three Types of Notes

| Type | Purpose | Example |
|---|---|---|
| **Fleeting notes** | Quick capture, disposable | "Idea: habit stacking for coding" |
| **Literature notes** | Summarize a source in your own words | "Atomic Habits argues that..." |
| **Permanent notes** | One atomic idea, fully developed | "The 1% Improvement Principle" |

## Workflow

1. **Capture**: Write a fleeting note when an idea strikes
2. **Process**: Convert fleeting notes to permanent notes (daily or weekly)
3. **Connect**: Link new permanent notes to existing ones
4. **Review**: Periodically review clusters in Graph View for insights
